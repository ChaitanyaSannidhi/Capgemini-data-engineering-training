Project Title

E-commerce Data Analysis using PySpark – Window Functions & Customer Analytics

Project Overview

This project focuses on analyzing an e-commerce dataset using PySpark in a Databricks/Lakehouse environment. The objective is to perform data cleaning, validation, and advanced analytical transformations to extract meaningful business insights related to customers, orders, and products.

The project demonstrates practical implementation of data engineering concepts, including ETL processing, referential integrity validation, window functions, and customer analytics, which are commonly required in real-world industry scenarios.

The dataset used resembles the Olist Brazilian E-commerce dataset, containing information about customers, orders, payments, products, and order items.

Objectives

The major objectives of this project are:

Load structured CSV datasets into PySpark DataFrames.
Perform data cleaning such as null handling and datatype conversions.
Validate referential integrity between datasets.
Apply window functions for advanced analytics.
Generate customer-level and product-level insights.
Implement customer segmentation logic for business understanding.

Data Processing Workflow
1. Data Loading

All CSV files are loaded into PySpark DataFrames using schema inference and header recognition.

2. Data Cleaning

Data cleaning ensures quality and reliability of analysis:

Removal of null values in important columns
Conversion of timestamp columns into proper datetime format
Filtering invalid or incomplete records
3. Referential Integrity Validation

Checks are performed to ensure relationships between datasets are valid:

Every order item must correspond to an existing order
Every order must be associated with a valid customer

This step ensures consistency similar to foreign key validation in relational databases.

Analytical Tasks Performed
Task 1: Top 3 Customers per City

Window functions are used to rank customers based on their total purchase amount within each city.

Business Insight:
Identifies high-value customers geographically.

Task 2: Running Total of Daily Sales

Cumulative sum of daily sales is calculated to observe revenue growth trends over time.

Business Insight:
Helps in understanding seasonal demand patterns and growth trajectory.

Task 3: Top Products per Category

Dense ranking is applied to identify top-performing products within each product category.

Business Insight:
Helps businesses focus on high-performing products.

Task 4: Customer Lifetime Value (CLV)

Total revenue generated by each customer across all their purchases is calculated.

Business Insight:
Identifies most valuable customers for retention strategies.

Task 5: Customer Segmentation

Customers are categorized into segments such as:

Gold – High value customers
Silver – Medium value customers
Bronze – Low value customers

Business Insight:
Useful for personalized marketing strategies and loyalty programs.
